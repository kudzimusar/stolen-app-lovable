## **What Happens When Reward Claims Are Rejected:**

### **1. Immediate Actions:**
- **Status Change**: Report status reverts from `contacted` ‚Üí `active`
- **Verification Reset**: Verification status changes to `unverified`
- **Notification Sent**: Owner receives email notification about rejection
- **Public Visibility**: Report remains visible on Community Board as "Lost"
- **Reward Status**: Changes from "Reward Pending" ‚Üí "Reward Available"

### **2. Owner Notifications:**
- **Email Notification**: "Your reward claim has been rejected"
- **Reason Provided**: Admin can specify rejection reason
- **Next Steps**: Guidance on how to improve claim validity
- **Appeal Process**: Information about how to appeal the decision

### **3. Community Impact:**
- **Report Visibility**: Device remains in "Lost" status
- **Community Engagement**: Others can still submit tips
- **Reward Availability**: Reward remains available for valid claims
- **Trust Score**: May affect owner's community trust rating

## **üìã Rules for Valid Reward Claims:**

### **Required Documentation:**
1. **Device Ownership Proof**:
   - Original purchase receipt
   - Device registration documents
   - Insurance policy details
   - Warranty information

2. **Identity Verification**:
   - Government-issued ID
   - Proof of address
   - Contact information verification
   - Social media verification (optional)

3. **Device Description Match**:
   - Serial number verification
   - Physical description accuracy
   - Unique identifiers (scratches, stickers, cases)
   - Device condition details

### **Claim Validation Process:**
1. **Initial Review** (Automated):
   - Document completeness check
   - Basic information verification
   - Duplicate claim detection

2. **Admin Review** (Manual):
   - Document authenticity verification
   - Cross-reference with police reports
   - Community feedback analysis
   - Risk assessment scoring

3. **Final Decision**:
   - Approve: Move to verification phase
   - Reject: Provide specific reasons
   - Request More Info: Additional documentation needed

---

## **üîÑ Rejection Reasons & Next Steps:**

### **Common Rejection Reasons:**
1. **Insufficient Documentation**:
   - Missing purchase receipt
   - Unclear device description
   - Incomplete contact information

2. **Suspicious Activity**:
   - Multiple conflicting claims
   - Inconsistent information
   - Fraud indicators detected

3. **Policy Violations**:
   - Claim submitted too quickly after loss
   - Device not properly registered
   - Previous fraudulent activity

### **Appeal Process:**
1. **Submit Additional Evidence**:
   - Additional documentation
   - Witness statements
   - Police report updates

2. **Admin Review**:
   - Secondary review by senior admin
   - Community feedback consideration
   - External verification if needed

3. **Final Decision**:
   - Appeal approved: Process continues
   - Appeal denied: Case closed

- **Be Responsive**: Answer admin questions promptly
